Coverage Criteria

本篇文章主要是把目前遇到的几种覆盖率收集起来，以方便将来研究过程中需要作比较

1.Stetement Coverage(语句覆盖)

又称为行覆盖（line coverage），段覆盖（segment coverage）, 基本块覆盖（basic block coverage）

语句覆盖目前来看是一种“最弱”的覆盖方式，它要求选择足够多的测试用例，使得程序中每个语句至少都能被执行一次。

对于该程序，我们只需要选择测试用例（a = 5, b = 5）作为输入就能达到100%的语句覆盖，但是这样的测试用例是无法检测出程序潜在的 除零错。显然一味追求语句覆盖率只能考虑到程序的大体结构，而对其中的逻辑关系并没有太多的关注。

2. Condition Coverage(条件覆盖)

条件覆盖关注的重点是每个子表达式的可能结果是否都已经考虑到，因此它要求每个判定语句中的 子表达式 的结果true和false都要被测试到

对于该程序，我们只需要选择测试用例（a = 5, b = 5）和（a = 15, b = 15）作为输入就能达到100%的条件覆盖。因为在这两个输入下第一个子表达式 a<10 的true和false的情况都出现了，同样的第二个子表达式 b<10 的两种情况都达到了。因此条件覆盖不是将判定中的每个子条件表达式的结果进行排列组合，而是只要每个子条件表达式的结果true和false都测试到即可

3. Decision Coverage(判定覆盖)

又称分支覆盖（branch coverage）, 所有边界覆盖（all-edges coverage）, 基本路径覆盖（basic path coverage），判定路径覆盖 (decision path coverage)

判定覆盖关注的重点在于判定语句（该语句可能含有多个子条件表达式）的结果

因此同样的例子，我们可以选择测试用例（a = 5, b = 15）和（a = 15, b = 15）作为输入来达到100%的判定覆盖，但是这样的测试用例却不能达到100%的条件覆盖。但是同样的，当我们选择测试用例（a = 5, b = 15）和（a = 15, b = 5）的时候能达到100%的条件覆盖，但是达不到100%的判定覆盖。

4. Path Coverage(路径覆盖)

又称断言覆盖（predicate coverage）

路径覆盖是在判定覆盖的基础上还考虑了不同判断语句的可能组合情况。目前来看是“最强”的覆盖方式

比如对于该例子，我们用测试用例（a = 5, b = 15）和（a = 15, b = 5）能达到100%的判定覆盖，但是这个程序的可能输出一共有（0，1，10，11）四种可能，而我们目前只发现了其中（1，10）两种可能。如果我们希望找出所有四种可能（100%的路径覆盖），那么我们需要生成四个测试用例（a = 5, b = 5），（a = 5, b = 15），（a = 15, b = 5）和（a = 15, b = 15）这样就保证了所有的路径都被探索到了。

5. MC/DC(Modified Condition/Decision Coverage)

MC/DC覆盖的关注点是希望每个子条件表达式（condition）的取值都能独立地影响到该判断语句（decision）的取值

假设我们当前的判断语句是 A and B, 那么它的完备测试用例集应该如下

用例组号 A B 结果
1 1 1 1
2 0 1 0
3 1 0 0
4 0 0 0

但是，考察上表中的第1、2组测试用例：条件A的所有取值均已出现且B的取值不变，同时结果的所有取值也均已出现，那么我们这时候就可以说条件A在条件B不变的情况下独立地影响判定的结果。

同理根据第1、3组测试用例，条件B也在条件A不变的情况下独立地影响判定的结果。

此时在这几个测试用例下，两个子条件表达式都能独立地影响到判断语句的结果，满足了MC/DC覆盖。因此我们综上分析可得出满足MC/DC覆盖的测试用例集如下：
